# Sales-Inventory-Management-System-SQL-

A relational database project that demonstrates the design and implementation of a **Sales Inventory Management System** using **MySQL**. The system efficiently manages customers, products, suppliers, orders, inventory, and payments while providing valuable business insights through SQL queries.

---

##  Project Overview

This project was developed to simulate a retail business database that supports inventory management, sales tracking, and reporting. It leverages SQL to organize business data, maintain data integrity, and generate analytical reports for informed decision-making.

---

##  Features

- Relational database design using SQL
- Customer, supplier, and product management
- Order and payment tracking
- Inventory monitoring
- Sales reporting and revenue analysis
- Customer purchase analysis
- Supplier performance evaluation
- Low-stock identification
- Business intelligence using SQL queries

---

##  Technologies Used

- **MySQL** – Database design and SQL querying
- **Microsoft Excel** – Data preparation and validation
- **PowerPoint** – Project presentation

---

##  Database Schema

The database consists of the following tables:

- Customers
- Suppliers
- Products
- Orders
- Order Details
- Payments

The schema uses **Primary Keys**, **Foreign Keys**, and integrity constraints to ensure accurate relationships between entities.

---

##  SQL Analysis Performed

The project includes SQL queries to analyze:

- Total sales and revenue
- Best-selling products
- Slow-moving products
- Customer purchase history
- Inventory levels
- Supplier performance
- Sales trends
- Order summaries

Queries utilize:

- JOINs
- Aggregate Functions
- GROUP BY
- ORDER BY
- Subqueries
- Filtering
